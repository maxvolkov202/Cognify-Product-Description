## Small-but-judgment (Max to pick)

### J1 — Zod errors leak to the client as stringified JSON
**Where:** `src/app/api/score/route.ts:199-206`, and mirrored in
`src/app/api/transcribe/route.ts` + `src/app/api/upload/route.ts`

Current behavior for invalid input returns:
```json
{"error":"invalid_input","message":"[\n  {\n    \"code\": \"too_small\", ..."}
```

That `message` field is a stringified Zod error array — fine for
debugging but ugly if anything client-side ever surfaces it. A simple fix
is to call `error.errors[0].message` and return the first human-readable
reason, and keep the raw dump under a `details` key for debugging.

**Fix shape:** 5-line change per route, zero product-facing risk. Not a
bug, but a polish item before the app sees real users.

### J2 — No end-to-end Deepgram smoke without a browser
**Where:** Test tooling, not product code.

The only way to verify the full transcription path (audio blob → word
timings → score) from the console is to record audio in a browser. I
validated the Deepgram API key directly against their `/v1/projects`
endpoint, so the key is live, but the handoff from our `/api/transcribe`
route to Deepgram is untested end-to-end.

**Fix shape:** A 15-line script that generates a 5-second tone via a
pure-JS WAV encoder (no ffmpeg dependency), POSTs it to
`/api/transcribe`, and asserts word-level timings come back. Nice-to-have
for the self-test loop; low priority.

## Bigger-thought (needs conversation)

### B1 — Scoring harshness calibration
**Observation:** The hedging rep got a composite of 27. That's
technically correct — the transcript *was* bad — but a composite below
30 might be too discouraging for a first-time user who's still learning
the feedback loop. The team's earlier spec said "Most reps will land
between 55 and 80" — this one fell well under that range.

**Tradeoff:** Real feedback beats kind feedback for advanced users, but a
brand-new user on their first rep who gets a 27 might bounce. A possible
middle: floor the composite at 40 for a user's first 3 reps, show the
real underlying dimension scores, and use the compressed composite only
for the "daily score" card. Or: add a rep-count-aware calibration pass.

**Why it matters:** This is the difference between a product that feels
like a coach (hard but fair, adapts to your level) and one that feels
like a critic (accurate but demoralizing). The current tuning leans
critic. Worth a Max decision before any real users see it.
