When presented with two ideas or approaches that could both have merit, merge them into the best possible version rather than picking one or keeping them separate. Lean toward doing more rather than less.

**Why:** Max explicitly stated *"any time between 2 ideas combine them into the best possible merge its better to do more"* during the Cognify v2 replan phase (April 2026). He prefers comprehensive, ambitious scope over conservative incremental changes. This is a general working style preference — not specific to one task or phase.

**How to apply:**
- When planning a phase or making a design decision, if two approaches both have merit, pick the merged / more-comprehensive version.
- When writing code, include adjacent improvements if they're natural to bundle in the same edit. Don't artificially split work into conservative chunks when a bigger coherent delivery is possible.
- When facing a split between "do X now, Y later" and "do X+Y together," prefer the combined option unless there's a specific reason to split.
- If the proposed scope feels too small, check whether it can be expanded into something adjacent that's also valuable.
- Caveat: don't use this as license to inflate scope with features the user didn't ask for. The principle applies to combining things already under consideration, not inventing new requirements.
